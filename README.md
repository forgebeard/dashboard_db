## 🚀 Быстрый старт

### 1. Подготовка окружения
Убедитесь, что установлен Python 3.11 или выше.

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Настройка переменных окружения
Создайте файл `.env` в корневой папке проекта:

```env
DB_USER=postgres
DB_PASSWORD=ваш_пароль_от_дампа
DB_HOST=localhost
DB_PORT=5432
```

### 3. Запуск
```bash
streamlit run src/app.py
```

## 📂 Структура проекта

Проект следует принципу разделения ответственности (SRP):

```text
src/
├── app.py                  # Точка входа: маршрутизация вкладок
├── config.py               # Глобальные настройки (лимиты, заголовки)
├── constants.py            # Справочники статусов (VM_STATUS_MAP, HOST_STATUS_MAP)

├── ui_utils.py             # Общие утилиты UI (например, фикс UUID)

├── vms/                    # Модуль Виртуальных машин
│   ├── vms_module.py       # UI: список, фильтры, инспектор
│   ├── vms_utils.py        # Логика: SQL-запросы, маппинги
│   ├── vms_diagnostics.py  # Просмотр сырых таблиц (vm_*)
│   └── vm_inspector_sql.py # Глубокий отчет по одной ВМ
│
├── hosts/                  # Модуль Хостов (аналогичная структура)
├── storage/                # Модуль Хранилищ
└── ...

## 📘 Руководство по модулям

### 🖥️ Хосты и 💻 ВМ
*   **Список:** Позволяет искать узлы по имени/FQDN, фильтровать по кластерам и выделять проблемные (не в статусе Up).
*   **Инспектор:** Генерирует текстовый отчет со статусами, ресурсами, сетью, дисками и последними событиями аудита.
*   **Диагностика:** Прямой просмотр системных таблиц (`vds_static`, `vm_dynamic`) с защитой от переполнения памяти.

### 🌐 Сети и 💾 Хранилища
*   Анализ логических сетей, vNIC профилей и состояния Storage Domains.
*   Проверка связей между хранилищами и дата-центрами.

### ⚡ Задачи и 📋 Аудит
*   Мониторинг зависших задач (Jobs) и анализ журнала событий (Audit Log) для поиска причин сбоев.

## ⚠️ Безопасность и ограничения

1.  **Конфиденциальность:** Дампы БД содержат чувствительные данные инфраструктуры. Храните их в защищенном месте.
2.  **Память:** При работе с таблицами `audit_log` или `event_history` всегда используйте фильтр по дате.
3.  **Локальный запуск:** Приложение не должно быть доступно из внешней сети. Используйте его только на локальном контуре (`localhost`).
